# 📊 Incident Analysis Dashboard

An end-to-end **Incident Management Analytics Project** built using **Python, Streamlit, and Power BI** to analyze IT Service Management (ITSM) data.

---

## 🚀 Live Demo

🔗 https://incident-analysis-dashboard-6wfwhkqddtiz5vxdmvdzyr.streamlit.app/

---

## 📌 Project Overview

This project transforms raw **incident event log data** into a structured dataset and builds interactive dashboards to generate actionable insights.

It helps analyze:

* 📈 Incident trends over time
* ⏱ SLA performance (Met vs Breached)
* 🔥 Priority distribution
* 📂 Top incident categories
* ⏳ Average resolution time
* 📊 Priority vs SLA relationship

---

## 🛠 Tech Stack

* Python (Pandas, Matplotlib, Seaborn)
* Streamlit (Interactive dashboard)
* Power BI (Advanced dashboarding)
* GitHub (Version control)

---

## 🔄 Workflow

1. Data Cleaning using Python
2. Data Transformation using Pandas
3. Feature Engineering (Resolution Time)
4. Visualization using Matplotlib & Seaborn
5. Dashboard Development:

   * Streamlit (Web App)
   * Power BI (Business Dashboard)

---

## 📊 Dashboard Features

### 🎛 Filters

* Category filter
* Priority filter

### 📈 Visualizations

* Incident Trend (Line Chart)
* SLA Distribution (Pie Chart)
* Priority Distribution (Bar Chart)
* Category Analysis
* Priority vs SLA Heatmap

### 📊 Key Metrics

* Total Incidents
* SLA %
* Average Resolution Time

---

## 🔍 Key Insights

* Majority incidents are **Moderate priority**
* SLA compliance is around **60%**
* Incident volume peaks in **March**
* High-priority incidents have higher SLA breach risk

---

## 📂 Project Files

* `incident_cleaning.py` → Data preprocessing
* `cleaned_incidents.csv` → Clean dataset
* `incident_dashboard.py` → Streamlit app
* `Incident_Analysis_Dashboard.pbix` → Power BI dashboard

## ▶️ How to Run Locally

```bash
pip install -r requirements.txt
streamlit run incident_dashboard.py
```
